Website Design — Build From Scratch

In this unit, you will design and publish a real website using HTML, CSS, and optional JavaScript.

You will not use website builders. Instead, you’ll learn how websites actually work — from files on your computer to a live site on the internet.

Why build from scratch?

Tools like Wix or Squarespace can build websites quickly, but they hide how things work. In this unit, the goal is understanding — not shortcuts.

  • You control your design
  • You can fix problems when they happen
  • You can move your site to new hosting later
  • You understand what’s happening behind the scenes

Not allowed in this unit:

  • Wix, Squarespace, Google Sites, Canva Websites
  • Pre-made site generators that hide the code

How websites work

A website is a system made of four parts:

  1. Files — HTML, CSS, images, and scripts created in VS Code
  2. Repository — GitHub stores your files and tracks changes
  3. Hosting — GitHub Pages publishes your site to the internet
  4. URL — the public link people use to visit your site

Update workflow:

Edit in VS Code → Save → Upload to GitHub → Commit → Live site updates

How this unit is organized

You’ll move through the unit using Stages. Each stage contains smaller Parts.

You may work at different speeds, but you should complete stages in order — each one builds skills you’ll need later.

Stages

Stage A — Roadmap + Plan

  • How websites work
  • Plan your site (purpose, pages, style guide)

Stage B — Coding Environment + First Page

  • Set up VS Code and project files
  • Create your first HTML page

Stage C — Build Pages + Use AI Thoughtfully

  • Write HTML, CSS, and optional JavaScript
  • Use AI as a coding assistant (optional)

Stage D — GitHub + Update Workflow

  • GitHub setup: account, repo, first upload
  • Updating your site without GitHub Desktop

Stage E — Publish + Multi-Page Navigation

  • Publish your site with GitHub Pages
  • Build a multi-page site with navigation

Stage F — Style, Content, Polish, Explain

  • Apply your style guide
  • Add content and images responsibly
  • Test, refine, and debug
  • Explain how your website works

What you will submit

  • Live website link (GitHub Pages)
  • GitHub repository link
  • Short explanation covering:
    • Where you wrote your code
    • How your files moved online
    • What GitHub does
    • What hosting does
    • How a custom domain could be added later
  • Evidence of your style guide in the design
