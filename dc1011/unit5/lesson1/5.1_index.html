5.1 Roadmap + Plan

Before you write any code, you need to understand how websites work and what you are building.

This stage gives you the big picture so the rest of the unit makes sense.

Part 1 — How Websites Work

A website is not one thing. It is a system made up of four connected parts.

  1. Website files
    HTML, CSS, images, and scripts stored in folders on your computer.
  2. Repository (GitHub)
    An online place that stores your files and keeps a history of changes.
  3. Hosting
    A computer on the internet that reads your files and shows them as a website.
  4. URL (address)
    The public link people use to visit your site.

How updates work:

Edit files in VS Code → Save → Upload to GitHub → Commit → Live site updates

This unit focuses on understanding each step — not using tools that hide what’s happening.

Important boundaries for this unit

  • Allowed: HTML, CSS, and optional JavaScript written by you
  • Allowed: GitHub and GitHub Pages for hosting
  • Not allowed: Wix, Squarespace, Google Sites, Canva Websites
  • Not allowed: One-click site generators that hide the code

The goal is not speed — it is understanding.

Part 2 — Plan Your Site

Before writing any code, you need a clear visual plan for your website. Planning first makes your site easier to build, easier to design, and easier to understand.

Professional designers don’t start by coding. They start by:

  • Looking at examples
  • Deciding on a visual style
  • Sketching or mocking up layouts
  • Refining their ideas before building

Step 1 — Explore & Get Inspiration

Spend a few minutes looking at real websites related to your topic or interests. You are not copying — you are collecting ideas.

Pay attention to:

  • Page layout and structure
  • Spacing and sections
  • Colour choices
  • Font styles
  • Overall vibe (clean, playful, bold, minimal, etc.)

Ask yourself: How does this site make me feel? Why?

Step 2 — Create a Visual Plan

Create a visual plan that shows what your website could look like. This does not need to be perfect or detailed.

Choose ONE option:

  • Sketch (paper): Draw a rough layout of your homepage. Show the header, navigation, main sections, and footer. Labels are enough.
  • Canva (recommended): Create a simple mockup or mood board that shows layout blocks, colour ideas, font styles, and overall vibe.

You are planning structure and style — not building the final site yet.

Step 3 — Revisit & Revise Your Style Guide

Now look back at your original style guide.

Ask yourself:

  • Do these colours still fit my idea?
  • Are these fonts readable for a website?
  • Does the visual tone match my site’s purpose?

You may revise your style guide if needed. Small changes are encouraged.

Design is iterative — refining your ideas is part of the process.
