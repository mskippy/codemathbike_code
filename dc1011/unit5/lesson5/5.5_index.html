```html DC 10/11 • 5.5 Publishing Your Website

Overview

Big idea: You can publish your website to the internet using GitHub (file hosting) + GitHub Pages (website publishing).

By the end of this lesson, you will be able to:

  • Choose a GitHub username and repo name that make a clean, shareable website URL.
  • Create a GitHub repository and upload a copy of your site files.
  • Publish your site using GitHub Pages and find your live link.

You should have completed: 5.1 Roadmap + Plan, 5.2 Website Content, 5.3 Setting Up Our Workspace, 5.4 Coding Our Website Pages.

Step 1 — Plan Your Website Identity

Before you publish, you need to make two choices. These choices become part of your public website address.

1.1 Choose a GitHub Username

Your GitHub username usually appears in your website URL:

https://USERNAME.github.io/REPOSITORY-NAME/

Username guidelines:

  • School-appropriate (you will share this link).
  • Easy to read and spell (no random characters).
  • Not personal info (avoid full legal names + birth years).

1.2 Choose a Repository Name (Your Site Name)

This is the second part of your website URL (after your username).

  • Use lowercase.
  • Use hyphens instead of spaces.
  • Keep it short and clear.

1.3 Write it Down

Before you continue, write down:

  • GitHub username: _______________________
  • Repository name: _______________________

Important: Email Address for GitHub

GitHub requires an email address to create your account.

  • You can use your school email.
  • Recommended: use a personal email you will still have after this course (easier for long-term maintenance).

Tip: Many people choose an email that matches their username (example: username alexcodes → email like alexcodes@...).

Step 2 — Create Your Repo & Upload Your Website

This is the standard workflow for this course. It works on school computers and keeps things simple.

2.1 Create a GitHub Account

  1. Go to github.com and sign in (or click Sign up).
  2. Create your account using the username you planned in Step 1.
  3. Verify your email if GitHub asks you to.

2.2 Create a New Repository

  1. Click the + icon (top right) → New repository.
  2. Repository name: use your planned site name (example: portfolio-site).
  3. Visibility: choose Public.
  4. Check Add a README file.
  5. Click Create repository.

2.3 Upload a Copy of Your Website Files

Your working files stay on your H drive. You are uploading a copy to GitHub for publishing.

  1. In your repo, click Add file → Upload files.
  2. Drag in all your site files and folders (example: index.html, css/, images/).
  3. Scroll down and click Commit changes.

2.4 Check Your Structure

Your repo should look like this (example):

index.html
css/
images/

Common mistake: don’t upload a folder that contains your website. Upload the contents of your website folder instead.

Level Up: Clone Your Repo

The standard workflow developers use to maintain a live website.

Cloning your repo links your local files to GitHub, allowing you to manage updates through commits instead of re-uploading files.

Cloning Your Repo (Optional)

What does “cloning” mean?
Cloning creates a linked copy of your GitHub repo on your computer. Instead of uploading files each time, you edit your files in the repo folder and then commit + push updates.

Use this if:

  • You feel confident with file management.
  • You want a more professional workflow.
  • You want easier long-term maintenance.

1 Install GitHub Desktop (If Allowed)

  1. Go to desktop.github.com and install GitHub Desktop.
  2. Open GitHub Desktop and sign in to your GitHub account.

2 Clone Your Repository

  1. In GitHub Desktop, choose File → Clone repository.
  2. Select your website repo.
  3. Choose a save location (H drive recommended).
  4. Click Clone.

3 Edit Inside the Repo Folder

From now on:

  • Open the cloned repo folder in VS Code.
  • Edit your files inside that folder.
  • Use GitHub Desktop to commit and push changes.

Very important: do not mix workflows. Choose either:

  • Upload workflow (Step 2A), or
  • Clone workflow (Step 2B)

Step 3 — Publish Your Website with GitHub Pages

3.1 Open GitHub Pages Settings

  1. Open your repository on GitHub.
  2. Click Settings.
  3. In the left menu, click Pages.

3.2 Turn On GitHub Pages

Under Build and deployment:

  • Source: Deploy from a branch
  • Branch: main
  • Folder: /(root)

Click Save.

3.3 Find Your Live Website Link

After a moment, GitHub will show your live URL. It should look like:

https://USERNAME.github.io/REPOSITORY-NAME/

3.4 Quick Test

  • Does your homepage load?
  • Do your nav links work?
  • Do your images and CSS load properly?

If it doesn’t work yet: wait 30–60 seconds and refresh. If you see a 404, double-check that index.html is in the main repo folder.

Checkpoint

Before you move on, make sure you have these saved somewhere you can easily find them:

  • Your live website link (GitHub Pages URL)
  • Your GitHub repo link

Next lesson (5.6), you’ll do a final polish + submit your link on Teams.

Quick links
  • github.com (sign in / repo)
  • Settings → Pages (publish)
  • desktop.github.com (optional Level Up)
Must-have files
  • index.html (required homepage)
  • css/ folder (styles)
  • images/ folder (assets)
Common fixes
  • 404 error: confirm index.html is in the root of the repo.
  • No CSS/images: check file paths + spelling (capital letters matter).
  • Updates not showing: refresh (Ctrl + Shift + R).
```
