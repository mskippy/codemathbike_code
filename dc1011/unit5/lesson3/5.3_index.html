Before you start: Where your website folder lives

At school, not all folders work the same way. For this unit, your website must live in your H: Drive so it’s accessible through the school file system.

Rule: If it’s part of your website, it lives here:

H: Drive
└── DigiComm1011
    └── your_project_folder

File naming rules: lowercase only • no spaces (use underscores) • keep names short

Step 1 — Create your project folder

  1. Open File Explorer and go to: H: Drive → DigiComm1011
  2. Create one folder for your website:
    • Lowercase
    • No spaces (use underscores)
    • Short and clear

    Examples: portfolio_site, music_blog, club_site

Help: If you don’t see your H: Drive
  • Restart File Explorer
  • Log out and back in
  • Ask the teacher for help (your drive may not be mounted)

Step 2 — Choose a navigation layout (and download your starter files)

You will choose one navigation layout and use it on every page of your site.

Important: Choose one layout and use it everywhere. Do not mix navigation styles.

Option A — Sidebar Navigation (Recommended)
  • Navigation links appear on the side
  • Content appears beside the navigation
  • Easier to keep consistent across pages
Option B — Navigation Under the Header
  • Header at the top
  • Navigation directly underneath
  • Clean and simple layout

Download your starter files using the links in the right rail. Save all files directly into your project folder.

  • You should end up with 5 files in your folder: 4 HTML pages + style.css
  • Do not rename anything yet

Extra inspiration (optional): Navbar Gallery

Step 3 — Open your folder in VS Code

  1. Open VS Code
  2. Go to File → Open Folder…
  3. Select your project folder (not individual files)

Rule: Open folders, not files. This keeps your website organized and prevents broken links.

Step 4 — Preview your website in a browser

Use one of these options:

  • Live Server (recommended)
  • Live Preview
  • Or double-click index.html to open it in a browser

Quick test:

  • If the page looks styled (not plain), your CSS link is working ✅
  • If it looks plain, something is missing or in the wrong folder ⛔

Workflow: change → save → refresh → check

Mini structure check (keep it simple)

Before you start typing a lot of content, make sure you know what each page area is for:

  • Header = site name/logo/tagline
  • Nav = links to your pages
  • Main = your actual page content
  • Sections = grouped chunks of content
  • Footer = credits / contact / socials (later)

Your Canva mock-up from 5.2 is your guide for what belongs on each page.

Step 5 — Start adding your content (this is the main work today)

Right now, everyone is using the same templates. That means websites will look similar at first — and that’s exactly what we want.

Your job today is to place your content into the correct sections. Styling comes next lesson.

✅ Safe to edit

  • Text between tags (example: <p>this text</p>)
  • Headings (<h1>, <h2>, <h3>)
  • Paragraphs and lists
  • The page title inside <title>

⛔ Do not edit (yet)

  • Navigation links inside <nav>
  • File names inside links (href="about.html")
  • Main structure tags (<header>, <nav>, <footer>)
  • CSS file links (href="style.css")
  • Layout structure (wrappers like .layout, <aside>, <main>)

Where most of your typing happens

<main>
  <h2>Page title</h2>

  <section>
    <h3>Section heading</h3>
    <p>Replace this text with your content from 5.2.</p>
  </section>
</main>

Use your Canva mock-up as a guide

  • Match your page sections to your Canva plan
  • Replace placeholder text with your real writing
  • If you need more sections, copy/paste an existing <section> block

Reminder: Don’t worry if it doesn’t look like Canva yet. Next lesson you’ll use CSS to format and customize.

Quick troubleshooting

My page looks plain (no styling)
  • Check that style.css is in the same folder as your HTML files
  • Make sure the CSS link in the template was not changed
  • Hard refresh the browser (Ctrl + Shift + R)
My links aren’t working / I get a “file not found” error
  • Make sure you did not rename files after linking
  • Check spelling: about.html is not the same as About.html
  • Keep everything lowercase
My changes aren’t showing up
  • Did you press Save in VS Code?
  • Did you refresh the browser?
  • Are you editing the correct file?
