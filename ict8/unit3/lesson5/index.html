Overview

The Heads Up Display (HUD) is where you display always-visible information like Score and Health. We’ll place these on a dedicated HUD layer so they render above gameplay and stay fixed on screen.

1) Create the HUD Layer

  1. Open your Level 1 scene and show the Layers Panel.
  2. Add new layer and rename it HUD<>.
  3. Drag the HUD<> layer to the top so it draws above all other layers.
  4. Make sure HUD<> is the active (selected) layer before adding HUD objects.

2) Add HUD Text Objects (BBText)

We’ll use BBText so we can format text and update it at runtime.

Score

  • New Object → BBText
  • Object Name: Score<>
  • BBCode Text: Score: 0<>
  • Font: Replace → Choose from Asset Store (e.g., Kenny High Square)
  • Base Size: ~50 (adjust to taste)
  • Base Color: Yellow (high contrast)

Health

  • Duplicate Score<> and rename to Health<>
  • BBCode Text: Health: 5<>
  • Base Color: Green

Place the objects on the layout:

  • Score: top-left
  • Health: bottom-left

3) Add a Global Score Variable

We’ll track score across scenes using a Global Variable:

  1. Open Project Manager → Project Settings → Global variables.
  2. Add Number variable PlayerScore<> with initial value 0<>.

Note: You already have PlayerHealth<> from earlier lessons.

4) Display Text from Variables

Add the following actions to your no conditions event at the top of your event sheet

  • Add action: Score<> → BBCode text
    • Modification's sign → = (set to)
    • Text → "Score : " + PlayerScore
  • Add action: Health<> → BBCode text
    • Modification's sign → = (set to)
    • Text → "Health : " + PlayerHealth

Keeping “display” actions grouped near the top makes it easy to maintain and guarantees the HUD refreshes every frame.

Don't forget to add spaces in between the quotes to make your text readable.

Now you can see your Player's Health and if it get hits by enemy fire, and now we can see the score when the enemy is killed.

Test & Polish

  • Run the game and destroy an enemy — confirm the score increases.
  • Get hit by an enemy bullet — confirm health decreases (from previous lesson logic).
  • Adjust font size/position for readability on your screen resolution.

Stretch: add a brief score pop-up animation near the enemy when destroyed.

Learning Goals

  • Create and order layers; keep HUD above gameplay.
  • Add BBText objects for Score and Health.
  • Use a Global Variable (PlayerScore<>) for cross-level scoring.
  • Bind text to variables with BBCode and update via events.

Tips

  • Lock gameplay layers to avoid moving HUD by accident.
  • Keep HUD text high-contrast; test at multiple resolutions.
  • Group “display” actions under a comment at the top of events for clarity.
