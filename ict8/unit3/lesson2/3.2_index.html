Overview

In this lesson, we’ll bring our game to life by adding a controllable player ship. You’ll learn how to create the player object, set up movement, keep the ship on screen, and make it fire bullets toward the mouse cursor. By the end, your game will have a player that can move, aim, and shoot — the foundation for all future gameplay.

Step 1 – Add the Player

To add the player, create a new Sprite object and configure behaviors.

  • Add a new object: Sprite
  • Object name: Player
  • Import image: player

Add these behaviors (in Behaviors):

  • Top Down movement — uncheck Rotate object
  • FireBullet

Press Apply, drag Player onto your scene, and preview to move the player. If it feels slow, increase Acceleration and Max speed. Save your work.

Step 2 – Resize the Player

Now it looks like the Player is too big for the space we are working it... Let's resize the player, but we need to do that in the events sheet.

  1. Open the event sheet → Add action under our first event.
  2. Add action (no condition): Player → Scale → 0.6.

Preview: the player should be smaller. Don't forget to save!

Step 3 – Keep the Player on Screen

Now you've probably noticed that the player goes off the screen. We need to add a behaviour to keep it ON screen

  • Double-click Player → Add behavior → Stay on Screen.
  • Set margins to 10 on all four sides.

Save and preview to confirm the player stays visible.

Step 4 – Firing Bullets

Great! Now we have a Player that moves around and stays on screen - but the bullets aren't shooting! Well there aren't any bullets to shoot! So we need to add a bullet and tell the game how to shoot the bullets from the Player.

    Adding the Bullet

  1. Create a new Sprite for the bullet:
    • Name: Bullet
    • Import image: laser_player.png
  2. Add behavior to Bullet: Destroy when outside of screen.

Now we need to add an event to make the bullets fire from our space ship towards where our cursor is on the screen.

    Add a new event

      Condition: Mouse button released → Left

      Action: Fire bullets towards a position

      • Object: Player
      • X: Player.X()
      • Y: Player.Y()
      • Bullet object: Bullet
      • Target X: CursorX()
      • Target Y: CursorY()

        • Preview: click to shoot bullets from the ship toward the cursor. Save your work.

Step 5 – Shoot from the Nose (Custom Point)

When we shoot the Bullets, they seem to be shooting from the wing. By default, bullets spawn at the origin. We can add a custom point and adjust our event so that the bullet shoots from the nose of the Player

  1. Main scene → double-click Player → Edit Points.
  2. Add a new point at the ship’s nose and name it ShootingPoint.
  3. Apply changes.

Now we need to update our code to say where we want the point to shoot from:

Go to your event sheet and click on the Player.X() to change it to the following

  • Click on Player.X()
  • In the expression editor, replace it with Player.PointX("ShootingPoint")
  • Do the same for Player.Y(), replacing it with Player.PointY("ShootingPoint")
  • Press OK to save your changes.
  • Preview: click to shoot bullets from the ship’s nose toward the cursor. Save your work.

Step 6 – Rotate Towards the Cursor

Great! Now we have a shooting Player, but the player doesn't rotate. We want the player to rotate towards where it is shooting, which in the direction of the cursor. To do this we will need to add an action.

  • Add an event (no condition)
  • Add action: Player → Rotate towards position
  • Set X to CursorX()
  • Set Y to CursorY()
  • Set Angular speed to 0
  • Press Apply

Preview: the ship should now rotate to face the cursor. Save your work.

Wrap Up

  • Player moves and stays on screen.
  • Bullets fire toward the cursor from the nose.
  • Ship rotates to face the cursor.
  • This is what your event sheet should look like

Skills Summary

  • Create and configure a Sprite object.
  • Add and adjust object behaviors (Top Down movement, FireBullet, Stay on Screen).
  • Resize sprites using the event sheet (Scale action).
  • Set up bullet objects with behaviors and firing events.
  • Use custom points (ShootingPoint) for precise bullet spawn locations.
  • Rotate the player object toward the cursor position.
  • Test and tweak movement speed, acceleration, and gameplay feel.
