Step 1 — Change the Browser Tab Title and Author (to Your Name)

This step updates the text that appears on your browser tab and the author information in the page’s metadata.

  1. Scroll to the very top of your HTML file in VS Code. You’ll see the title and author lines near the top.
  2. Change only the placeholder text to your first name + last initial (for example, Alex M).

    Do not delete any tags, angle brackets, or quotation marks—only replace the name text inside them.

  3. Save your file (Ctrl + S) and refresh your Live Server preview (or reload your browser tab).

Step 2 — Update the Main Heading (to Your Name)

This step updates the large heading at the top of your page (the “Hi, I’m …” line).

  1. Locate the main heading in your HTML file in VS Code.
  2. Change only the name text inside the highlighted area (keep the tags and any emoji).

    Do not delete any angle brackets or tag names—only replace the placeholder name with your first name + last initial (e.g., Alex M).

  3. Save (Ctrl+S) and refresh your Live Server preview (or reload your browser tab).
  4. Quick check: The heading should read “Hi, I’m Your Name …”.
    • If it didn’t change, make sure you saved and refreshed.
    • If the line looks broken, undo once or twice (Ctrl+Z) and re-select only the name text.

Step 3 — Write Your “About Me” (3–4 Sentences)

This step replaces the placeholder text in the About Me section with your own short paragraph.

  1. Find the About Me section in your HTML file in VS Code.
  2. Select only the paragraph text (the words), not the tags or angle brackets.

    Do not delete any tags or angle brackets. Change only the words inside the paragraph.

  3. Write 3–4 sentences about yourself:
    • Interests or hobbies (one or two)
    • Something you’re proud of or a fun fact
    • One goal for this course or what you want to learn

    Privacy: Use first name + last initial only. Keep it school-appropriate.

  4. Save (Ctrl+S) and refresh your Live Server preview (or reload the browser tab).
  5. Quick check:
    • 3–4 complete sentences, easy to read
    • No last names, addresses, or personal contact info
    • Spelling/grammar looks good
Having trouble pasting from Word?

If your text looks weird after pasting from Word, try this: paste into Notepad first, copy again, then paste into VS Code to remove extra formatting.

Step 4 — Add 2–3 Real Links You Use

Replace the placeholder links with 2–3 websites you actually visit (learning, hobbies, school tools, etc.). Use clear labels like “Khan Academy – Algebra” instead of “click here”.

  1. Find the Links section in your HTML file in VS Code.
  2. Select only the link label and URL to change them.

    Do not delete any angle brackets, tag names, or quotation marks. Change only the visible link text and the URL between quotes.

  3. Add 2–3 links you actually use.
    • Choose useful, school-appropriate sites.
    • Make the label specific (e.g., “Unity Learn – 2D Beginner Pathway”).
    • Avoid pasting huge raw URLs as the label—keep them as the clickable text’s destination only.
  4. Save (Ctrl+S) and refresh your Live Server preview (or reload the browser tab).
  5. Test each link: click to make sure it opens the correct site in your browser.
    • If a link doesn’t open, double-check that you copied the full URL (including https://).
    • If you accidentally broke the line (weird colours/brackets), press Ctrl+Z to undo and re-select only the text and URL, not the tags.
Tips (optional)
  • Use short, descriptive labels—help your reader know why the link matters.
  • Prefer official sites (.edu, .org, vendor docs) when possible.
  • Keep it tidy: two or three quality links are better than ten random ones.

Step 5 — Add One Image + a Short Credit

Choose a school-appropriate image, show it on your page, and include a short credit below it.

  1. Pick an image source (safe + allowed).
    • Your own photo/graphic, or
    • A free/Creative Commons image (e.g., Unsplash, Pexels, Wikimedia Commons).
  2. Copy the direct image link (the file itself).
    • Right-click the image → Copy image address.
    • Direct image links usually end in .jpg, .png, or .gif.
    • If it ends with weird stuff (no file extension), try a different image or click “Open image in new tab” first.
  3. Paste the image address into the image spot in your HTML.

    Change only the URL and the alt text. Don’t delete any tags, angle brackets, or quotes.

  4. Write a meaningful alt text (1 short sentence).
    • Describe the image for someone who can’t see it (e.g., “Green game character jumping over a platform”).
    • Don’t write “image of…”—just describe it.
  5. Add a short credit line directly under the image.
    • Example: “Image credit: Author/Website (License).”
    • If it’s your own image: “Image credit: Your Name.”
  6. Save (Ctrl+S) and refresh your Live Server preview.
  7. Quick check
    • Image loads quickly and looks appropriate.
    • Alt text is short and descriptive.
    • Credit line includes source/author (and license if noted).
Troubleshooting
  • Image not showing? Re-copy the image address (not the page URL). Try opening the image in a new tab to get a clean link.
  • Broken layout? Undo once or twice (Ctrl+Z) and make sure you only edited the URL and alt text—no tags or brackets.
  • Too large? Pick a smaller image or a different size. Big files load slowly.

Step 6 — Make the Pun Button Your Own

Customize the joke line, the button text, and the message that appears when you click the button.

  1. Find the joke/pun area in your HTML file (heading + sentence + button together).
  2. Edit the joke setup text (the one visible above the button).

    Change only the words. Don’t delete tags or angle brackets.

  3. Change the button label to something fun but clear (e.g., “Tell me!”, “Reveal punchline”, “Make me groan”).
  4. Update the punchline/message (what appears after clicking).

    This text is usually stored right below the button or inside a small script area (you’ll see quotation marks around it).

    Change the text inside the quotes only. Keep the quotes and punctuation. If you see a semicolon ;<>, leave it in place.

  5. Save (Ctrl+S) and test it in your Live Server preview.
    • Click the button — the message should appear/change below the joke.
  6. Quick check
    • Setup sentence is school-appropriate.
    • Button label makes sense.
    • Punchline appears when clicked.
Troubleshooting
  • Nothing happens on click? Make sure you’re testing in the browser (Live Server), not the VS Code preview pane.
  • Error or weird colours in VS Code? You might have deleted a quote or angle bracket. Press Ctrl+Z to undo and re-edit only the text between quotes.
  • Message shows the old text? There may be a second punchline text in the script area—scroll a little further and update that one too.

Step 7 — Change the Page Accent Colour

Update the site’s main accent colour (used for buttons, links, and highlights) in the style area near the top of your file.

  1. Find the style/settings area near the top of your HTML (where colours are defined).
  2. Click the accent colour value to open the colour picker, then choose your new colour.

    Edit only the colour value. Keep punctuation (like : and ;) and any variable/property names intact.

  3. Save (Ctrl+S) and refresh your Live Server preview (or reload the browser tab).
  4. Quick contrast check
    • Buttons/links are easy to read against the background.
    • Avoid very light pastels on white backgrounds.
    • Make sure visited links are still distinguishable.
Optional: Add a background image with a readability overlay (simple steps)
  1. Choose a wide, subtle image (at least 1920×1080), preferably your own or from a free/CC source. Keep the file under ~1–2 MB.
  2. Copy the direct image address (the file URL that ends with .jpg or .png).
  3. In your style/settings area, find the background image setting and replace only the URL in quotes with your image address.
  4. Ensure the background is set to fill nicely: use settings that make it cover the screen, center it, and prevent repeating.
  5. Add a semi-transparent overlay colour above the image to keep text readable. Start around 30–50% opacity and adjust until headings and body text are clear.
  6. Save and refresh. If text is hard to read, increase the overlay opacity or choose a simpler image.

Step 8 — Complete Your Course Reflection

Use the existing Course Reflection section. You’ll replace the placeholder text with your own reflection and (optionally) add one more paragraph for depth.

  1. Find the Course Reflection section in your HTML file in VS Code.
  2. Replace the placeholder text with your reflection.

    Edit only the words inside the paragraph(s). Don’t delete any tags or angle brackets.

    • Write 3–5 sentences about: what you learned, a challenge you solved, and a takeaway or next goal.
    • Use first person (“I learned…”, “I improved at…”).
    • Keep it school-appropriate; first name + last initial only.
  3. Optional: add a second paragraph for an example or goal.
    • Put the cursor at the end of your first paragraph, press Enter, and type a short second paragraph (2–3 sentences).
    • Example focus: one specific skill you improved (e.g., HTML structure) or one goal for next time.
  4. Save (Ctrl+S) and refresh your Live Server preview.
  5. Quick check
    • Reflection is clear, in your own words, and easy to read.
    • Paragraphs are separated (not one giant block).
    • Spelling and punctuation look good.
Tips (optional)
  • Start with a strong lead: “This term I learned how to…”.
  • Mention one challenge + how you solved it.
  • End with a specific next step (“Next, I want to learn…”).

Step 9 — Finishing Touches: Footer Name + Date

Personalize the small line at the bottom of your page and make sure the date is correct.

  1. Find the footer area at the bottom of your HTML in VS Code.
  2. Change the name to your first name + last initial.

    Edit only the words—don’t delete any tags or angle brackets.

  3. Update the date to today (or confirm it’s already correct).
    • Use a clear format (e.g., Nov 4, 2025 or 2025-11-04).
  4. Save (Ctrl+S) and refresh your Live Server preview.
  5. Quick polish pass
    • Scroll the whole page—spacing looks good; headings are consistent.
    • Links and buttons use your accent colour and are easy to read.
    • Image has alt text + a short credit underneath.
    • Pun button works and shows your custom message.
    • About Me + Course Reflection read clearly (no last names/contact info).
Optional: Make the year update automatically (advanced)
  1. Decide where you want the automatic year (e.g., “© Your Name YEAR”).
  2. Add a tiny script that inserts the current year into that spot.
  3. Save and refresh—confirm the year matches this year; it will update each January.

If you want, ask your teacher to provide the ready-made snippet so you can paste it in safely.
