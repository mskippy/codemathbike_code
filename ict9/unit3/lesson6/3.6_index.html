Overview

Big idea: Build a multi-page promotional website for your game using VS Code and Microsoft Copilot, and apply your branding consistently across all pages.

In this lesson you will:

  • Create at least three pages for your game website.
  • Use Microsoft Copilot in your browser to help you write HTML and CSS.
  • Apply your logo, colours, fonts, poster, banner, and screenshots across the site.
  • Organize your content so visitors can easily understand and get excited about your game.

You should have completed: 3.2 Game Branding • 3.3 Style Guide • 3.4 Poster & Banner Design • 3.5 Game Trailer Production (draft or final).

Website Creation Basics (Using Microsoft Copilot + VS Code)

You will build your website in VS Code and use Microsoft Copilot in your browser to help you write and improve your HTML/CSS. Copilot is a helper — you are still the designer.

1. Set up your website folder

Create a folder structure for a multi-page site:

GameWebsite
│
├── index.html          (Home / Overview / Trailer)
├── gallery.html        (Screenshots / Poster / Banner)
├── how_to_play.html    (Controls / Goals / Tips)
├── style.css           (shared styling)
└── images
      ├── logo.png
      ├── banner.png
      ├── poster.png
      ├── screenshot1.png
      ├── screenshot2.png
      └── ...
  1. Create this structure on your H: drive or OneDrive.
  2. Put your logo, banner, poster, and screenshots inside the images folder.
  3. Open the GameWebsite folder in VS Code.

You can choose different page names (for example about.html), but you must have at least three HTML pages that all share the same style.css.

2. Plan your site structure

Before asking Copilot for code, decide what each page will do. A simple structure:

  • index.html – Home
    • Navigation bar
    • Hero section (banner, logo, tagline)
    • Short “About the Game” section
    • Trailer section (embedded video or placeholder)
  • gallery.html – Gallery
    • Screenshots
    • Poster and/or banner
    • Short captions for images
  • how_to_play.html – How to Play & Credits
    • Controls and main goal
    • Tips for success
    • Your name(s) and any asset credits

Quickly sketch or list where your trailer, poster, and key information will go.

3. Use Copilot for your home page (index.html)
  1. Open Microsoft Copilot in your browser (for example in Edge).
  2. Ask for a simple, beginner-friendly home page. For example:
"Create an HTML5 home page for my 2D platformer game.

It should include:
- a navigation bar with links to Home, Gallery, and How to Play
- a hero section with a banner image (images/banner.png), game title, and tagline
- an About the Game section with a heading and placeholder paragraph
- a Trailer section with a heading and a placeholder for an embedded video

Link to a CSS file called style.css.
Use simple, clean HTML that a beginner can understand."
  1. Copy Copilot’s HTML into index.html in VS Code.
  2. Save and open index.html in your browser to preview.
  3. If the code is too complex, ask Copilot: “Rewrite this HTML to be simpler and more beginner-friendly.”
4. Use Copilot for your other pages

Repeat the process for gallery.html and how_to_play.html.

Gallery page prompt example:

"Create an HTML gallery page for my game.

Include:
- the same navigation bar as my home page (Home, Gallery, How to Play)
- a heading 'Gallery'
- a grid of 4 images using files:
  images/screenshot1.png, images/screenshot2.png,
  images/screenshot3.png, images/poster.png
- short captions under each image

Link to style.css and keep the HTML simple."

How to Play page prompt example:

"Create an HTML page called How to Play for my game.

Include:
- the same navigation bar as my other pages
- sections for: Controls, Game Goal, Tips, and Credits

Use clean HTML and link to style.css."

Copy the code into the correct file in VS Code, then test that the navigation links move between all three pages.

5. Ask Copilot for shared CSS (style.css)

Once your pages and navigation are working, use Copilot to help with design.

CSS prompt example:

"Write a CSS file called style.css for my multi-page game website.

Requirements:
- Use a sans-serif font
- Background colour: #F2F7FF
- Primary colour: #2264F8
- Accent colour: #F97316
- Style the navigation bar so links are in a row, with spacing and hover styles
- Limit the main content width to about 900px and center it on the page
- Add spacing between sections and make images responsive

Keep the CSS simple and suitable for a beginner."
  1. Paste the CSS into style.css in VS Code.
  2. Refresh your pages in the browser to see the changes.
  3. If something doesn’t match your style guide, ask Copilot to adjust the CSS (for example, darker headings, larger h1, more padding).
6. Your responsibility (not Copilot’s)
  • Choose page names and sections that make sense for your game.
  • Make sure all pages share the same navigation and branding.
  • Rewrite text so it sounds like your voice, not an AI.
  • Check that every link and image works correctly.
  • Ask Copilot to explain or simplify code you don’t understand.

Activity A – Plan Your Multi-Page Site

  1. Confirm your site map.
    At minimum:
    • index.html – Home / Overview / Trailer
    • gallery.html – Screenshots / Poster / Banner
    • how_to_play.html – Controls / Goal / Tips / Credits
  2. Decide where each asset will go.
    Answer:
    • Which page will show the trailer?
    • Where will your poster and banner appear?
    • Which page will list your controls and credits?
  3. Check your navigation.
    Make sure that every page includes the same navigation bar with working links to Home, Gallery, and How to Play.

Activity B – Build & Style Your Pages

  1. Set up headings and content on each page.
    • Add clear headings (for example: Gallery, How to Play, Credits).
    • Write your About and How to Play text in your own words.
    • List your controls and game goal clearly.
  2. Add images and media.
    • Insert your logo, poster, banner, and screenshots using correct file names.
    • Place your trailer or a “Trailer coming soon” placeholder on the home page.
  3. Refine the layout with CSS.
    Use style.css to adjust:
    • Font families and sizes.
    • Colours that match your style guide.
    • Spacing (padding/margins) between sections.
    • Image sizes so nothing looks stretched or squished.

    If you get stuck, you can paste a small part of your CSS into Copilot and ask it to: 

    "Improve this CSS to center the content, add more spacing, 
and keep it simple for a beginner."

Activity C – Branding & Quality Check

Use this checklist to review your site before handing it in:

  • Pages: You have at least three HTML pages.
  • Navigation: All pages share the same navigation bar and links work.
  • Branding: Logo, colours, and fonts match your style guide.
  • Assets: Poster, banner, screenshots, and trailer (or placeholder) appear somewhere on the site.
  • Text: All writing is in your own words and makes sense.
  • Layout: Content is readable, spaced out, and not crowded.

Make any final changes using VS Code, and use Copilot only when you need suggestions or improvements.

Hand-In Checklist

When you are ready to submit your website in Teams, make sure:

  • Your GameWebsite folder contains all of your pages, style.css, and the images folder.
  • All links and images work when you open the site in your browser.
  • Your branding is consistent and your pages match the requirements of this lesson.

Your teacher will let you know whether to upload a ZIP folder or share a OneDrive / H: drive link for your GameWebsite folder.
