Overview

This lesson is about exploring. You’ve built a working platformer with enemies, stomp-to-defeat, pickups, and a HUD. Now try some common features and start thinking about your own original game idea.

Complete at least three challenges. Experiment, test, and take notes—you’ll use what you learn when you design your own game in the next lessons of this unit.

These are suggestions on how you can carryout each challenge, you can customize to your liking.

Challenges

  1. Spend time creating your platforms

    Polish your level geometry: add varied ledge heights, gaps, safe zones, and optional routes. Try one “skill check” jump that teaches timing.

  2. Add points for killing enemies

    When the player stomps an enemy, add score += 5 (or your value) and update the HUD. Keep pickup points separate so you can tune balance later.

  3. Add new enemies & change their characteristics

    Create a second enemy type or tweak variables (speed, patrol width, jump strength, damage). Consider a “tough” enemy that takes two stomps (use an hp object variable).

  4. Add new levels that get harder

    Make a Level 2 scene and increase difficulty gradually (enemy count, spacing, tighter jumps). Add a Next Level trigger at the end of Level 1.

  5. Add a health component to your player

    Create a health scene or object variable (e.g., 3). On side/below enemy hit, health -= 1, flash briefly, and show hearts or a health number on the HUD. Go to Game Over when health reaches 0.

  6. Start & Game Over screens

    Build a simple Title scene (Play/Quit). Build a Game Over scene that appears when lives/health reach 0. Add buttons to restart or return to Title.

  7. Restarting the game

    Add a key (e.g., R) to restart the current scene, or auto-restart after Game Over. Also restart if the player falls off the level.

  8. Level decorations (polish)

    Add background details and other elements to polish your game. Use the tilemap to add more items and be mindful of the Z-order of instances.

  9. Your own idea

    Try moving platforms, springs, checkpoints, health pickups, timers, or a quick mini-boss. Keep scope small and test often.

Reflection

  • Which three challenges did you implement? What worked right away, and what required debugging?
  • How did your changes affect difficulty and fun? Give one specific example (e.g., “+5 points on stomp made riskier routes feel rewarding”).
  • Identify one mechanic you’d keep for your own game and one you’d cut. Why?
  • Write a 2–3 sentence pitch for your own game: theme, main goal, and the 2–3 mechanics you’ll include first.

Helpful Hints

  • Score on stomp: on top-hit event, score += 5; keep HUD text in a blank-condition action.
  • Enemy HP:
    add object variable hp; on stomp, hp -= 1; delete when hp <= 0.
  • Health/Lives:
    health = 3; on side hit, health -= 1; if 0 → change scene to Game Over.
  • Next Level: add a goal object; on collision → Change to scene (Level 2).
  • Restart: key R → Restart the current scene.
