Intro

Now it’s time to add an enemy. In this lesson you’ll create a simple enemy that patrols back and forth on a platform using a per‑enemy state variable ("left"/"right"). In the next lesson we’ll add stomp to defeat and player damage.

  • Enemy sprite — an animated character that is auto‑controlled (no keyboard input).
  • Variables — values stored on an object (or scene) that control behaviour.
  • Collisions — when one object comes into contact with another.

1. Add the Enemy Object

  1. Add a new object → Sprite → name it Enemy. Select Import images.
  2. Select enemyWalking_1.png and enemyWalking_2.png from assets → PNG → Enemies from your U2 Game Development folder.
  3. In the Enemy Editor name this Animation #0 Walk. Loop the animation and change the timing to 0.225.
  4. Edit points → select Origin and set approximately center‑bottom (example for a 32×48 sprite: X 16, Y 44). Click Apply.
  5. Add the Platformer Character behaviour:
    • Disable default keyboard controls.
    • Set Max speed to 140–180 (start with 140).
    • (Leave gravity as default so the enemy stays grounded.)
  6. Place one or two Enemy instances on top of platforms in the scene.

2. Per‑Enemy Variable (Direction)

  1. Open the Enemy object → Variables → Add variable:
    • Name: state
    • Type: String
    • Default value: "right"
  2. (Optional) In the scene, select one Enemy instance and set its Instance variable state to "left" so it starts moving left. This shows that each enemy can behave independently.

3. Creating Edge Markers

Before we can animate the Enemy we need to set up some "Markers" that tells the Enemy to turn around and go the other way. These markers will also keep the enemy at the top of platforms so they don't fall off the edge.

  1. Add a new Tiled Sprite object. Name this object Right Edge Marker
  2. Select Create with Piskel

    This is where you can create your own custom images

  3. Keep the size at 32 x 32, change the colour to a Red and select the paint bucket to fill the area. Save your sprite.

    Why Red? Red = Right

  4. Repeat the steps to create a Left Edge Marker using a Lime colour.
  5. Save your work.

    6. This is the list of objects you should have by now. If you have extra - delete them and keep your files organized. Now we are ready to animate our Enemy!

4. Enemy Animations

First thing we need to do is to put Edge markers to define the enemy area. Choose an area, up on a plateau or between two "walls". Add your Left and Right markers, just inside the platform area. Add your Enemy between the two markers.

Go to your Level 1(Events) sheet and enter the following events:

    This set of events animates the Enemy's movement back and forth

    Add a new event

    • Condition : Enemy → Variables → Object variable value → state = "right"
    • Action : Enemy → Platformer controls → Simulate right key press
    • Action : Enemy → Effects → Flip the object horizontally → Activate flipping → NO

    Add a new event

    • Condition : Enemy → Variables → Object variable value → state = "left"
    • Action : Enemy → Platformer controls → Simulate left key press
    • Action : Enemy → Effects → Flip the object horizontally → Activate flipping → YES

    This set of events tells the Enemy to change directions when it hits a marker

    Add a new event

    • Condition : Enemy → Collision → Collision → Right_Edge_Marker
    • Action : Enemy → Variables → Change object variable value → state = "left"

    Add a new event

    • Condition : Enemy → Collision → Collision → Left_Edge_Marker
    • Action : Enemy → Variables → Change object variable value → state = "right"

    Save your work and preview. You sould now see your enemy moving back and forth along the platform.

    Place some more markers and enemies along your platforms. Save and preview your work.


    Hiding Markers

    To hide all your edge markers, add an action in the event with a blank condition. Add the following:

    • Action : Left_Edge_Marker → Visibility → Hide
    • Action : Right_Edge_Marker → Visibility → Hide

    Save all your work and preview.

    Your event sheet should look like this

5. Preview & Save

  • Preview and confirm each Enemy moves according to its state ("right" or "left").
  • Adjust Max speed if movement feels too slow/fast.
  • Confirm facing: moving right = not flipped; moving left = flipped. Swap the flip actions if your art faces the opposite way.
  • Place a second Enemy with state = "left" to confirm each instance moves independently.
  • Save your project (Ctrl+S).

6. Reflection

  • In 2–3 sentences, explain how the state variable controls Enemy movement and why a text value ("left"/"right") keeps events readable.
  • If an Enemy doesn’t turn at a marker, list two fixes you’d try (e.g., move marker inward; check the collision event; disable default controls; confirm the state comparisons).
  • If you couldn’t use markers, how else could you make a patrolling enemy (hint: ledge/wall checks or raycasts)? What are the trade‑offs?
