Intro

Now it is time to add the player. We will import a multi‑animation Player sprite, configure its hitbox/origin, add the Platformer Character behavior, and create events to switch between Idle, Run, Jump, and Fall animations in GDevelop.

  • Player sprite = animated character you control.
  • Origin & collision = where the object is “held” and how it collides with platforms.
  • Platformer Character = built‑in behavior for gravity, jump, and movement.

1. Add the Player Object

  1. Add a new object → Sprite → name it Player. Select Import images.
  2. Select playerRed_stand.png from U2 Game Development → PNG → Players → Player Red folder.
  3. In the Player Editor name this Animation #0 Idle.
  4. Click + Add an animation to add another animation for Walk.
  5. Click + Add a sprite and select playerRed_walk1.png, playerRed_walk2.png, and playerRed_walk3.png.
  6. Repeat Step 4 for the following animations
    • Jump : playerRed_up1.png, playerRed_up2.png, and playerRed_up3.png
    • Fall : playerRed_fall.png
  7. For the Walk and Jump we need to do a bit more work. If we preview the animation, it doesn't look right.
    • First check the Loop box. This repeats the animation.
    • Press the Preview, you'll notice that the animation is moving very fast. Change the FSP to 5, this will slow the animation down a bit.
    • The animation still looks weird - it's missing some images. When the animation loops it goes through the images from left to right, then back to the left image again. So it is missing an image inbetween. Select the second image and duplicate it. Make sure it goes at the very right. Preview the animation.
    • Repeat for the Jump animation.

      • This is what your Player Properties should look like

  9. Click on Edit points and click on the Origin
    • Set X to 19.5 and Y to 48. Press Apply

      • This will help keep the player inline with the platforms and other objects

  10. DO NOT press Apply - we need to add some behaviours first!

2. Add Platformer Character Behavior

  1. Select the Behaviors tab → Add a behavior → choose Platformer Character.
  2. Leave all the values as is for now, we can adjust later
  3. Confirm your ground objects have the Platform behavior and any pass‑throughs use Jump‑thru Platform.
  4. Press Apply. Now we can add our player to the scene.
  5. Drag your Player from the object menu and place it above a platform.
  6. Preview your game and see if you can move your Player

3. Setting the Camera

In your preview, you probably noticed that the Player runs off the screen! In order to fix that, we need to set up the camera so that it follows the Player.

  1. Double click on the Player object and select the Behaviour tab.
  2. Add the Smooth Camera and Smooth platformer camera and Apply those behaviours.
  3. Save and Preview your work and you should see the camera follow your player.

Now that we have that going, you probably noticed that there is some deadspace that we don't want to see. Let's fix that by enforcing some boundaries so that our camera only shows what we want the user to see. To do this we will need to Add an event.

  1. Click on your Level 1(Events) sheet
  2. Add a new event
    • Add condition : Leave this blank
    • Add action : Enforce Camera Boundaries

      Other actions → Camera → Layers and cameras → Enforce camera boundaries

      • Left = 0
      • Top = 0
      • Right = 6400
      • Bottom = 768
      • Layer = Base layer
      • Camera = 0

        The bottom number is the number of squares in the height of the viewport times 64- so that the platform is at the bottom of the viewing area.

      • Save your work and preview your game. The camera should now follow your player.

4. Player Animations

To make our player more interesting, we want to add the animations to our player movement. We have already added the animations to our Player, but now now we need to tell our program what to do with it. We can do this in our Level 1(Events)sheet.

  1. Add the Jump animation.

    Add a new event

    • Condition : Player → Platform state → Is Jumping
    • Action : Player → Animations and images → Animation (by name) set to → Jump

  2. Add the Fall animation.

    Add a new event

    • Condition : Player → Platform state → Is Falling
    • Action : Player → Animations and images → Animation (by name) set to → Fall

  3. Add the Walk animation.

    Add a new event

    • Condition : Player → Platform state → Is Moving AND
    • Add condition : Player → Platform state → Is On Floor
    • Action : Player → Animations and images → Animation (by name) set to → Walk

  4. Add the Idle animation.

    Add a new event

    • Condition : Player → Platform state → Is On Floor AND
    • Add condition : Player → Platform state → Is Moving and click on Invert condition

      Here we want to say "When the player is NOT moving" - that is what invert means

    • Action : Player → Animations and images → Animation (by name) set to → Walk

  5. Save your work and preview. But now we notice that the Player doesn't look in the direction it is moving. We need to add a couple more events.

    Add new event

    • Condition : Key is pressed → Left
    • Action : Player → Effects → Flip the object horizontally and select Yes to activate flipping.

    Add new event

    • Condition : Key is pressed → Right
    • Action : Player → Effects → Flip the object horizontally DO NOT select YES.
  6. Save all your work and preview.

    7. Your event sheet should look like this

    The yellow bar is a Comment that you can add to organize your code.

5. Z Order

We will be adding other sprites in the way of enemies and pickups. If we add them AFTER the Player they will appear on top of the Player. This is because on the scene, the Player instance has a lower Z value because it was added first. Think of Z-order as the layers in a layer.

To ensure your Player is on top of other sprites and objects on the Base layer we need to edit the Z value

  1. Click on the Player that is in your scene
  2. The Properties Panel for the Instance-Playershould open.
  3. Change the Z value to 100 (or some high number)

    4. Now it will be on top of any sprites added to the layer

6. Preview & Save

  1. Click Preview and test: move left/right, jump, watch animations change.
  2. Fix issues:
    • Falls through platforms? Ensure platforms have the correct behavior and the player’s feet hitbox is flat.
    • Wrong animation? Check event order and names.
    • Facing wrong way? Verify FlipX logic on key press.
  3. Save your project (Ctrl + S).
